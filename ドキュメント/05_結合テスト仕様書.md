# 05 結合テスト仕様書

## 1. テスト方針

### 1.1 目的
バックエンドAPIとフロントエンド間の連携、ならびにSupabaseとの疎通を検証する。

### 1.2 テスト環境
- バックエンド: ローカル起動 (`cargo run`)
- フロントエンド: ローカル起動 (`npm run dev`)、または本番ビルド
- データベース: Supabase（開発用プロジェクト）
- ツール: curl / ブラウザ DevTools

### 1.3 前提条件
- Supabase の全テーブル・インデックス・トリガーが作成済み
- `backend/.env` に正しい環境変数が設定済み
