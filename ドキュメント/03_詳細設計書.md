### 1.2 モジュール別設計

#### config.rs
- `Config` 構造体: `SUPABASE_URL`, `SUPABASE_KEY`, `EMAIL_HMAC_SECRET`, `SERVER_PORT`, `CORS_ORIGIN`
- `Config::from_env()` で環境変数から一括ロード

#### db.rs — SupabaseClient
- `select(table, query)` → GET `/rest/v1/{table}?{query}`
- `insert(table, body)` → POST `/rest/v1/{table}` + `Prefer: return=representation`
- `update(table, id_filter, body)` → PATCH `/rest/v1/{table}?{id_filter}`
- `delete(table, filter)` → DELETE `/rest/v1/{table}?{filter}`
- `new()` でベースURLを正規化: 末尾スラッシュ除去 + `/rest/v1` 未付加なら自動付加
- 全メソッドで `apikey` / `Authorization: Bearer` ヘッダーを付与

#### errors.rs — AppError
| バリアント | HTTPステータス |
|-----------|--------------|
| Unauthorized | 401 |
| NotFound(msg) | 404 |
| Conflict(msg) | 409 |
| BadRequest(msg) | 400 |
| Internal(msg) | 500 |

#### middleware/auth.rs
1. Cookie `session_token` を取得
2. TBL_SESSION を `sestk=eq.{token}` で検索
3. `sesea > now()` を確認（期限切れなら 401）
4. `sesui` から AuthenticatedUser { id } を構築
5. `X-Requested-With` ヘッダー検証（POST/PUT/DELETE）

### 1.3 サービス層詳細

#### auth_service.rs
- `signup(db, username, email, password)`:
  1. メールアドレスを正規化してHMAC-SHA256ハッシュ化
  2. 同一ハッシュがTBL_USERに存在すれば 409
  3. Argon2idでパスワードハッシュ化
  4. TBL_USERにINSERT
- `signin(db, email, password)`:
  1. メールアドレスをハッシュ化してTBL_USERを検索
  2. `verify_password`でパスワード照合
  3. 256bit乱数トークン生成、TBL_SESSIONにINSERT（有効期限1年）
  4. セッショントークンを返す
- `signout(db, token)`: TBL_SESSIONからDELETE

#### goal_service.rs
- 有効ステータス: `["exam_date", "passed", "failed", "abandoned"]`
- デフォルトステータス: `"exam_date"`
- `create`: `goash` (勉強時間) フィールドも含めてINSERT
- `update`: ステータスが `"passed"` に変更された場合、レスポンスに `promote_to_holding: true` を含む
- `study_hours` は 0.5h 単位（フロント側でバリデーション、バックエンドはf64として受け入れ）

#### community_service.rs
- RPC関数なし。4本の PostgREST クエリで集計:
  1. TBL_USER: 自分以外の全ユーザー取得
  2. TBL_HOLDING + TBL_MASTER join: `select=holui,holdt,TBL_MASTER(masnm)`
  3. TBL_GOAL + TBL_MASTER join: `select=goaui,goast,goash,goatd,TBL_MASTER(masnm)`
  4. TBL_FAVORITE: 現在ユーザーのお気に入り一覧
- HashMap で集計後、ページネーション適用

## 3. ビルド・デプロイ構成

### 3.1 Dockerfile（マルチステージ）

```
Stage 1: node:20-slim
  → npm install && npm run build
  → 成果物: /app/frontend/dist/

Stage 2: rust:latest
  → cargo build --release
  → 成果物: /app/target/release/shikaku_toruzoo

Stage 3: debian:bookworm-slim
  → ca-certificates インストール
  → バイナリ + dist/ をコピー
  → PORT 8080, EXPOSE 8080
  → CMD ["/app/shikaku_toruzoo"]
```

### 3.2 環境変数

| 変数名 | 説明 |
|--------|------|
| SUPABASE_URL | Supabase プロジェクトURL（/rest/v1 自動付加） |
| SUPABASE_KEY | service_role キー（RLSバイパス） |
| EMAIL_HMAC_SECRET | メールアドレスHMACシークレット（32バイト乱数推奨） |
| CORS_ORIGIN | フロントエンドオリジン（Cloud RunサービスURL） |
| SERVER_PORT | リッスンポート（デフォルト: 8080） |
| RUST_LOG | ログレベル（デフォルト: info） |
| STATIC_DIR | 静的ファイルディレクトリ（デフォルト: /app/static） |
