# 01 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
資格取るぞー！アプリ

### 1.2 目的
ユーザーが保有する資格情報を一元管理し、取得を目指す資格への目標設定・勉強時間の記録を行うことで、自己成長を支援するWebアプリケーションを構築する。他ユーザーの状況可視化やお気に入り機能によるモチベーション維持も実現する。

### 1.3 スコープ

| # | 機能 |
|---|------|
| 1 | 独自認証によるユーザー管理（サインアップ / サインイン / サインアウト） |
| 2 | 所持資格の登録・一覧・編集・削除 |
| 3 | 目標の設定・管理（ステータス・勉強時間・目標日） |
| 4 | 資格名マスタの自動蓄積と入力補完 |
| 5 | コミュニティ（他ユーザーの資格・目標・勉強時間の可視化） |
| 6 | お気に入り機能 |
| 7 | Google Cloud Run へのデプロイ |

### 1.4 対象外スコープ

- Supabase Auth の利用（独自認証を使用する）
- 認可・ロールベースアクセス制御
- フォロー / フォロワーなどのSNS的関係管理
- メール通知・プッシュ通知
- 管理者画面

## 2. 機能要件

### FR-001: ユーザー認証

| 項目 | 内容 |
|------|------|
| ID | FR-001 |
| 機能名 | ユーザー認証 |
| 概要 | メールアドレスとパスワードによるサインアップ・サインイン・サインアウト |
| 詳細 | ・サインアップ時にユーザー名・メールアドレス・パスワードを入力<br>・メールアドレスはHMAC-SHA256、パスワードはArgon2idでハッシュ化してDB保存<br>・セッションはサインアウトしない限り1年間保持<br>・サインアウト時にセッションを無効化 |
| 優先度 | 必須 |

### FR-002: 所持資格管理

| 項目 | 内容 |
|------|------|
| ID | FR-002 |
| 機能名 | 所持資格管理 |
| 概要 | 自身が所持する資格の登録・一覧表示・編集・削除 |
| 詳細 | ・資格名は入力補完付きテキストフィールドから入力<br>・取得日を登録可能<br>・ダッシュボード上でカード形式で一覧表示<br>・各資格の編集・削除が可能 |
| 優先度 | 必須 |

### FR-003: 目標設定・管理

| 項目 | 内容 |
|------|------|
| ID | FR-003 |
| 機能名 | 目標設定・管理 |
| 概要 | 取得を目指す資格への目標設定・勉強時間記録・ステータス管理 |
| 詳細 | ・資格マスタから資格を選択し、目標日を設定<br>・ステータス：受験日 / 合格 / 不合格 / 断念<br>・勉強時間を0.5h単位で記録（+/-ボタン操作）<br>・メモを1000文字以内で記録可能<br>・「合格」に変更した際、所持資格へ自動追加するか確認 |
| 優先度 | 必須 |

### FR-004: 資格マスタ管理

| 項目 | 内容 |
|------|------|
| ID | FR-004 |
| 機能名 | 資格マスタ管理 |
| 概要 | 資格名のマスタデータベースの自動蓄積と入力補完 |
| 詳細 | ・新規資格名入力時はマスタDBに自動追加<br>・次回以降の入力時にマスタからの候補をサジェスト（2文字以上で発動、300msデバウンス）<br>・カテゴリ（IT・語学・金融・法律・その他）で分類<br>・正規化した名前で重複登録を防止 |
| 優先度 | 必須 |

### FR-005: コミュニティ

| 項目 | 内容 |
|------|------|
| ID | FR-005 |
| 機能名 | コミュニティ |
| 概要 | 他ユーザーの資格・目標・勉強時間をダッシュボード上で可視化 |
| 詳細 | ・自分以外の全ユーザーを一覧表示<br>・各ユーザーの所持資格一覧・目標一覧（資格名/ステータス/勉強時間）を表示<br>・合計勉強時間を表示<br>・合格実績のあるユーザーにグッドマークを表示<br>・お気に入りユーザーを一覧上部に優先表示 |
| 優先度 | 必須 |

### FR-006: お気に入り機能

| 項目 | 内容 |
|------|------|
| ID | FR-006 |
| 機能名 | お気に入り機能 |
| 概要 | 他ユーザーをお気に入り登録し、コミュニティ一覧上部に優先表示 |
| 詳細 | ・各ユーザーカードに☆マークを配置<br>・クリックでお気に入りのトグル（☆⇔★）<br>・お気に入りユーザーはコミュニティ一覧の最上部に表示 |
| 優先度 | 必須 |

## 3. 非機能要件

### セキュリティ

| 項目 | 内容 |
|------|------|
| パスワードハッシュ | Argon2id（メモリ硬化型） |
| メールアドレスハッシュ | HMAC-SHA256 + サーバー管理シークレットキー |
| セッション管理 | 256bit暗号学的乱数トークン、HttpOnly/Secure/SameSite=Strict Cookie |
| 通信 | HTTPS（Cloud Runマネージド証明書） |
| CORS | フロントエンドオリジンのみ許可 |
| レート制限 | 認証: 5回/分、その他: 60回/分 |
| ソースマップ | 本番ビルドで無効化 |

### パフォーマンス

| 項目 | 内容 |
|------|------|
| API応答 | 95パーセンタイルで500ms以内 |
| 初回ロード | 3秒以内（LTE環境） |

### 可用性

| 項目 | 内容 |
|------|------|
| 稼働率 | 99.5%以上 |
| 最小インスタンス | 1（コールドスタート回避） |
| 最大インスタンス | 10 |

## 4. 制約事項

- バックエンドは Rust (Actix-web 4) で実装する
- フロントエンドは Vue 3 + TypeScript + Bootstrap 5 で実装する
- データベースは Supabase（PostgreSQL）を REST API 経由で使用する
- デプロイ環境は Google Cloud Run とする
- 全機能をシングルページのダッシュボードで提供する

## 5. 用語定義

| 用語 | 定義 |
|------|------|
| 資格マスタ | 全ユーザー共通の資格名称データベース |
| 所持資格 | ユーザーが既に取得済みの資格 |
| 目標 | ユーザーが今後取得を目指す資格・期限・勉強記録 |
| グッドマーク | 目標ステータスが「合格」の実績があるユーザーに付与されるバッジ |
| お気に入り | 他ユーザーをブックマークする機能 |
