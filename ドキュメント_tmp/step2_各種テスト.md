| Vue/TypeScript | Vitest + Vue Test Utils | Vitestの高速テストランナーを活用 |

### 1.3 カバレッジ目標
- バックエンド: 行カバレッジ 80% 以上

# 第2部 結合テスト仕様書

## 1. テスト方針

### 1.1 対象範囲
- バックエンドAPI（ハンドラ → サービス → DB連携の一連の流れ）
- フロントエンド画面（コンポーネント → ストア → API呼び出しの一連の流れ）

### 1.2 テスト環境
- バックエンド: Actix-web の test_server を使用。Supabaseはテスト用プロジェクトを使用
- フロントエンド: Vitest + MSW（Mock Service Worker）でAPIレスポンスをモック

### 1.3 テストデータ
- テスト開始時にテスト用DBをクリーンアップし、Seedデータを投入
- Seedデータ: ユーザー3名、資格マスタ10件、所持資格・目標を各ユーザーに複数件
